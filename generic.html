Match It AR

Match It AR is a match two game mechanic but, in a 3D, augmented reality game. The augmented part of the game is the play area where the interaction with the game objects takes place. Within this play area, the player matches targets much like in Bejeweled, by lining them up in a row of two or more, but in this case the line-up is done using a raycast from the camera. Once the lined-up targets are destroyed, new ones are spawned in the scene at new positions.

The game was developed in Unity 3D and used ARCore for the AR elements of the game. The AR scanning was done using the ARCore. I followed steps from Google’s codelabs introduction to ARCore to set up the tracking session. ARCore uses motion tracking and environmental understanding to scan flat surfaces and track features. I then look for any Touch input and check if it hits any of the tracked planes, if so then I create an anchor at that position. The player can match targets. There’s a ray being cast every frame to check for targets so the player doesn’t have to take any action like touching a button. If the ray hits a target then it checks everything else the ray is hitting. If the first two hits have the same colour then that makes a match. Once they’re matched they get destroyed and respawned using the initial spawning code. Three arrays of floats, hard coded, store values that are used to give the target a position and no targets share the same position.

This game formed part of my undergraduate project which looked at aspects of UX design in mobile AR games. The code repository, full text, and presentation powerpoint are available below.
